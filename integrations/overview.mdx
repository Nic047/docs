Mac SDK is designed to be extensible. You can add support for any macOS application by extending the `BaseApp` class and registering it with the `AppManager`.

```typescript
import { Mac, AppManager } from '@mac-sdk/automation';
import { BaseApp } from '@mac-sdk/automation/core/base-app.js';
```

<Note>
  `BaseApp` is not exported from the main `@mac-sdk/automation` entry point. Import it directly from the subpath as shown above.
</Note>

## Built-in integrations

Mac SDK ships with built-in support for:

<Columns cols={2}>
  <Card title="Apple Notes" icon="note-sticky" href="/api-reference/overview#notes-app">
    Create, search, and manage notes with Markdown support.
  </Card>
  <Card title="VS Code" icon="code" href="/api-reference/overview#vs-code-app">
    Open files, folders, workspaces, and scaffold projects.
  </Card>
</Columns>

Any other application can be controlled through the generic `mac.app()` interface, which provides lifecycle methods like `activate()`, `quit()`, `isRunning()`, and more.

```javascript
const safari = mac.app('Safari');
await safari.activate();
console.log('Is running:', safari.isRunning());
```

## Creating custom integrations

Extend Mac SDK with your own application integrations by subclassing `BaseApp`:

```typescript
import { Mac } from '@mac-sdk/automation';
import { BaseApp } from '@mac-sdk/automation/core/base-app.js';

class SlackApp extends BaseApp {
  constructor() {
    super('Slack');
  }

  async sendMessage(channel: string, message: string) {
    // Use AppleScript to interact with Slack
    await this.activate();
    // Add your custom automation logic here
  }
}

// Register the integration
const mac = new Mac();
mac.apps.register('slack', new SlackApp());

// Use it
const slack = mac.app('slack');
await slack.sendMessage('#general', 'Hello from Mac SDK!');
```

### Available base methods

When you extend `BaseApp`, your custom app inherits these methods:

| Method | Description |
| --- | --- |
| `activate()` | Bring the app to the foreground |
| `quit()` | Quit the application |
| `isRunning()` | Check if the app process is running |
| `hide()` | Hide the app window |
| `show()` | Show/unhide the app |
| `minimize()` | Minimize the front window |
| `wait(ms?)` | Pause for a duration (default: 500ms) |
| `getWindows()` | Get titles of all open windows |
| `isFrontmost()` | Check if the app is the frontmost process |

### Internal helpers

Inside your custom app class, you can use these methods to interact with the system:

| Method | Returns | Description |
| --- | --- | --- |
| `_executeAppleScript(script)` | `string` | Execute an AppleScript string (15s timeout) |
| `_shellCommand(cmd)` | `string` | Execute a shell command (10s timeout) |
| `_escapeAppleScript(script)` | `string` | Escape single quotes for AppleScript |
| `_escapeString(str)` | `string` | Escape quotes and newlines for embedding in scripts |

```typescript
import { BaseApp } from '@mac-sdk/automation/core/base-app.js';

export class SpotifyApp extends BaseApp {
  constructor() {
    super('Spotify');
  }

  async getCurrentTrack() {
    const script = `
      tell application "Spotify"
        return name of current track & " - " & artist of current track
      end tell
    `;
    return this._executeAppleScript(script);
  }

  async playPause() {
    const script = `tell application "Spotify" to playpause`;
    return this._executeAppleScript(script);
  }
}
```

## Managing app instances

Use the `AppManager` (accessible via `mac.apps`) to register and manage your integrations:

```javascript
import { Mac } from '@mac-sdk/automation';

const mac = new Mac();

// Register custom apps
mac.apps.register('spotify', new SpotifyApp());
mac.apps.register('slack', new SlackApp());

// Access them
const spotify = mac.app('spotify');
const track = await spotify.getCurrentTrack();

// List running apps
const running = await mac.apps.getRunningApps();
console.log('Running apps:', running);

// Kill an app
await mac.apps.killApp('Safari');
```

## Error handling

All `BaseApp` methods throw standard `Error` objects when AppleScript or shell commands fail. Wrap calls in try/catch blocks:

```typescript
try {
  const spotify = mac.app('Spotify');
  await spotify.activate();
} catch (error) {
  console.error('Failed to activate Spotify:', error.message);
}
```

<Note>
  Some methods like `hide()`, `show()`, and `getWindows()` require **Accessibility permissions**. Grant access in **System Settings** > **Privacy & Security** > **Accessibility** (macOS 13+) or **System Preferences** > **Security & Privacy** > **Privacy** > **Accessibility** (macOS 12 and earlier).
</Note>
