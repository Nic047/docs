## Prerequisites

Before installing Mac SDK, ensure you have the following:

- macOS 10.15 (Catalina) or later
- Node.js 14.0.0 or later
- Xcode Command Line Tools (install with `xcode-select --install`)

<Note>
  Mac SDK is an ES module package (`"type": "module"`). Your project must use ES module syntax (`import`/`export`). Add `"type": "module"` to your `package.json` or use the `.mjs` file extension.
</Note>

## Verifying installation

Create a test file `test.js`:

```javascript test.js
import { Mac } from "@mac-sdk/automation";

async function main() {
  const mac = new Mac();

  // Test basic functionality
  await mac.notification("Installation Successful", "Mac SDK is ready to use!");
  console.log("Installation verified successfully!");
}

main().catch(console.error);
```

Run the test:

```bash
node test.js
```

## Permissions

Some Mac SDK features require macOS permissions. Grant them in **System Preferences** > **Security & Privacy** > **Privacy**:

| Permission | Required for |
| --- | --- |
| **Accessibility** | `hide()`, `show()`, `minimize()`, `getWindows()`, and UI scripting |
| **Screen Recording** | `screenshot()` |
| **Automation** | Controlling other applications via AppleScript |

Add your terminal app (Terminal, iTerm, VS Code, etc.) to each of these categories as needed.

## Troubleshooting

### Permission errors

If you encounter permission errors, grant the necessary permissions:

<Steps>
  <Step title="Open System Preferences">
    Go to **System Preferences** > **Security & Privacy** > **Privacy**.
  </Step>
  <Step title="Add your terminal app">
    Add your terminal app to **Accessibility**, **Screen Recording**, and **Automation**.
  </Step>
</Steps>

### Module not found

If you see `Cannot find module '@mac-sdk/automation'`:

1. Verify the package is installed:
   ```bash
   npm list @mac-sdk/automation
   ```
2. Check your `package.json`:
   ```json
   "dependencies": {
     "@mac-sdk/automation": "^0.1.0"
   }
   ```

### ERR_REQUIRE_ESM

If you see `ERR_REQUIRE_ESM`, Mac SDK uses ES modules and cannot be loaded with `require()`. Either:

1. Add `"type": "module"` to your `package.json`.
2. Rename your script file to use the `.mjs` extension.

### TypeScript errors

If you are using TypeScript, ensure you have the latest type definitions:

```bash
npm install --save-dev @types/node
```

Mac SDK includes its own TypeScript definitions in `types/index.d.ts`.
